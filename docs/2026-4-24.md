# 前端&AI技术双周刊-2026.04.24


## 前端技术
- [Bun v1.3.12](https://bun.com/blog/bun-v1.3.12)
  <br>Bun v1.3.12 内置无头浏览器自动化与 Bun.cron。Bun 继续把“运行时 + 工具链”做成一体，这版开始提供原生、无头的浏览器自动化能力（方便做 E2E、抓取、渲染任务），同时 Bun.cron 让你在进程内直接跑定时任务。

- [Intl API](https://polypane.app/blog/the-intl-api-the-best-browser-api-youre-not-using/)
  <br>Intl API 可能是你没充分使用的最强浏览器 API 之一。Intl 不只是日期格式化：数字、货币、相对时间、排序（collation）等都能直接用。文章以代码为主，适合快速把“国际化该怎么做”从依赖库转回到浏览器原生能力。

- [OWASP NPM 安全最佳实践清单](https://cheatsheetseries.owasp.org/cheatsheets/NPM_Security_Cheat_Sheet.html)
  <br>OWASP 发布了持续更新的 NPM 安全最佳实践清单，涵盖禁用生命周期脚本、防范 typosquatting 攻击、可信发布和依赖混淆防护等关键议题。对于维护 Node.js 项目供应链安全具有重要参考价值。

- [Node.js 24.15.0 LTS 发布](https://nodejs.org/en/blog/release/v24.15.0)
  <br>Node.js 24.15.0 LTS 版本正式将 require(esm) 和模块编译缓存标记为稳定特性，新增 --max-heap-size 标志和 OpenSSL 4.0 构建支持。require(esm) 的稳定化意味着 CJS 和 ESM 互操作性问题得到官方解决，开发者可更自由地混用两种模块系统。

- [SVG Filters 入门指南——从滤镜区域到颜色插值](https://frontendmasters.com/blog/svg-filters-guide-getting-started-with-the-basics/)
  <br>把 SVG 滤镜当成一套“可组合的图像处理流水线”来理解会更顺

- [AEO（Agentic Engine Optimization）让站点更容易被智能体发现与使用](https://addyosmani.com/blog/agentic-engine-optimization/)
  <br>SEO 关注“被人搜索到”，而 AEO 更像是“被 AI 智能体读懂并能可靠调用”。Addy Osmani 给出一组更偏工程的建议：结构化内容、清晰的动作与结果、稳定的页面信息架构，以及让自动化工具更容易抽取关键字段——听起来像“为机器写文档”，但对面向 AI 助理的产品越来越现实。

- [MDN 放弃 React 重构前端架构](https://developer.mozilla.org/en-US/blog/mdn-front-end-deep-dive/)
  <br>MDN 重建了整个前端技术栈，放弃 React 转向 Web Components 和自研服务端组件系统。这一决策展示了如何为内容密集型网站构建现代前端，减少不必要的 JavaScript 发送量，是对"默认使用框架"思维的一次重要反思。

## AI 资讯
- [OpenAI正式推出GPT-5.5](https://openai.com/index/introducing-gpt-5-5/)
  <br>2026 年 4 月 24 日，OpenAI 正式推出其迄今最强大的 AI 模型——GPT-5.5，不仅在智能水平上实现质的飞跃，更以惊人的能效比重塑大模型经济性边界。该模型专为“智能体”（Agent）时代设计，能在模糊指令下自主规划、调用工具、执行多步骤复杂任务，在代码生成、科研探索与企业自动化等领域表现卓越。

- [阿里巴巴 ATH 推出开放世界模型 Happy Oyster，开启实时交互体验申请](https://www.happyoyster.cn/)

## 扩展阅读
- [垂直代码库组织架构实践](https://tkdodo.eu/blog/the-vertical-codebase)
  <br>Dominik Dorfmeister 主张放弃按 components/hooks/utils 等技术类型的水平分层，转向按业务领域的垂直组织。以 Sentry 代码库为例，水平结构导致单个 components 目录膨胀至 200+ 文件。核心原则是"一起变更的代码应住在一起"，并建议用 eslint-plugin-boundaries 等工具强制执行模块边界。
