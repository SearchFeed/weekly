## 前端技术
- [Next.js 16.1](https://nextjs.org/blog/next-16-1)
  <br>Next.js 16.1 带来了更快的开发工作流和更高的稳定性，主要更新包括 Turbopack 文件系统缓存的稳定化、实验性的 Bundle Analyzer 工具以及更便捷的调试功能。Turbopack 文件系统缓存显著提升了开发服务器的启动速度，特别是在大型项目中。新的 Bundle Analyzer 帮助开发者优化代码包大小，提升核心 Web 指标。此外，Next.js 16.1 还改进了对 `serverExternalPackages` 的处理，减少了配置负担。

- [2025 JavaScript Rising Stars](https://risingstars.js.org/2025/en)

## 拓展阅读
- [MicroQuickJS (MQuickJS) - A JavaScript Engine for Embedded Systems](https://github.com/bellard/mquickjs/blob/main/README.md)
  <br>MicroQuickJS（MQuickJS）是一个专为嵌入式系统设计的轻量级JavaScript引擎，仅需10 kB的RAM和约100 kB的ROM即可运行。它支持ES5子集，并采用严格模式以避免低效或易出错的JavaScript特性。MQuickJS通过紧凑的垃圾回收机制和优化的内存管理，显著降低了资源占用，同时保持了与QuickJS相当的性能。对于需要在资源受限环境中运行JavaScript的开发者来说，这是一个极具价值的工具。

- [How to 'Steal' Any React Component](https://javascriptweekly.com/issues/768)
  <br>探讨了如何通过 React 内部数据结构（Fiber）和 LLM 技术重现生产环境中的 React 组件，对于组件分析和逆向工程有参考价值。

- [Stop Turning Everything Into Arrays (and Do Less Work Instead)](https://javascriptweekly.com/issues/768)
  <br>介绍了迭代器助手方法，这是一种更高效的数据处理方式，能够减少不必要的数组转换，提升代码性能。

- [npm to Implement Staged Publishing After Turbulent Shift Off Classic Tokens](https://socket.dev/blog/npm-to-implement-staged-publishing)

- [JavaScript's for-of Loops Are Actually Fast](https://javascriptweekly.com/issues/768)
  <br>澄清了 for-of 循环性能方面的误解，帮助前端工程师更好地选择循环方式。
