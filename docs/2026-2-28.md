## 前端技术

- [Node.js v25.7.0 (Current)](https://nodejs.org/en/blog/release/v25.7.0)
  <br>Node.js 25.7.0 发布，新增 HTTP/2 的 HTTP/1 回退配置 http1Options，SEA 支持 ESM 入口，SQLite 进入 RC 阶段，并改进 test runner 在 SIGINT 下的中断提示。同时升级 npm 11.10.1、llhttp 9.3.1 等依赖，优化 Stream、ESM Loader、内存与压缩处理，修复 HTTP、Worker、FileHandle 泄漏等问题。

- [Node.js 24.14.0 (LTS)](https://nodejs.org/en/blog/release/v24.14.0)
  <br>Node.js 24.14.0（LTS）发布，该版本带来多项功能新增与优化，涉及 async_hooks、fs、http、sqlite 等多个模块，还升级了 npm 等依赖、优化了构建配置与测试体系，同时修复了各类功能漏洞并完善文档

- [Vuetify v4](https://v4.vuetifyjs.com/en/)
  <br>Vuetify v4 是基于 Vue 3 的最新主要版本，全面支持 Material Design 3，优化了包体积和 TypeScript 类型定义，为开发者提供更现代、高效的 UI 组件库体验。

- [TypeScript 6.0 Beta 发布](https://devblogs.microsoft.com/typescript/announcing-typescript-6-0-beta/)
  <br>这是 TS 最后一个基于 JavaScript 构建的版本，旨在为 7.0 版本的 Go 语言底层重写做铺垫（预计带来 10 倍编译提速）。6.0 版本默认开启 strict: true，默认 target 升级为 es2025，并全面拥抱 ESM，原生支持 Temporal API。

- [Firefox 148 推出 setHTML API](https://hacks.mozilla.org/2026/02/goodbye-innerhtml-hello-sethtml-stronger-xss-protection-in-firefox-148/)
  <br>Firefox 148 引入了新的 setHTML() API，旨在替代 innerHTML。该 API 默认内置了更强的 XSS 防护机制，能自动清理恶意脚本，帮助开发者更安全地操作 DOM，减少安全漏洞。

- [React Native 0.84 ](https://reactnative.dev/blog/2026/02/11/react-native-0.84)
  <br>React Native 0.84 版本正式发布，HermesV1默认提速；iOS预编译加快构建；移除旧架构；需Node.js22；同步React19.2.3、ESLintv9；增强图片、无障碍与URL，优化了渲染性能并修复了关键问题

## 拓展阅读
- [Oxfmt Beta](https://oxc.rs/blog/2026-02-24-oxfmt-beta)
  <br>Oxfmt Beta 发布，作为基于 Rust 的 Prettier 兼容格式化工具，首轮无缓存运行速度号称比 Prettier 快 30 倍。新版已通过 100% JS/TS 一致性测试，支持多种前端文件格式，内置 Tailwind、Import 与 package.json 排序，并提供一键迁移与 Node.js API，向稳定版迈进。

- [Fastest Frontend Tooling for Humans & AI](https://cpojer.net/posts/fastest-frontend-tooling)
  <br>文章指出 2026 年是 JS 工具链全面提速的一年，重点推荐用 Go 重写的 tsgo 大幅提升 TS 类型检查速度，以 Oxfmt、Oxlint 替代 Prettier 与 ESLint，并搭配 @nkzw/oxlint-config 严格配置，同时优化 npm-run-all2、ts-node 等开发体验，仍保留 pnpm、Vite、React 作为核心工具，旨在打造无妥协、反馈极快、规范严格的开发环境。

- [Electrobun v1.基于Bun的桌面应用程序方法](https://blackboard.sh/blog/electrobun-v1/)
  <br>介绍JS/TS桌面应用程序的跨平台运行时。它使用操作系统的默认web渲染器（如Neutralinojs），Bun作为引擎，幕后使用bundler。

- [CodeMirror 审查工具](https://aziis98.com/blog/codemirror-review-tool/)
  <br>介绍了作者基于 CodeMirror 开发的一款代码审查工具，探讨了如何利用 CodeMirror 的 API 实现代码差异对比和交互功能
