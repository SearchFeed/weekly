## 前端技术
- [pnpm 11.3 发布——支持 pnpm stage、trustLockfile、原生 pkg/repo 命令](https://pnpm.io/v/11.3.0)
  <br>pnpm 11.3 新增 staged publishing 支持（pnpm stage）、trustLockfile 校验增强及原生 pkg/repo 子命令，与 npm Staged Publishing 特性配套，适合 monorepo 项目采纳。

- [Deno 2.8 发布——Node.js API 兼容率达76.4%，新增 audit fix / pack / why，内置 V8 14.9](https://deno.com/blog/v2.8)
  <br>Deno 2.8 将 Node.js API 兼容率提升至76.4%（超 Bun），新增 deno audit fix / deno pack / deno why 等命令补全工程链，取消安装包时 npm: 前缀要求，并随 V8 14.9 带来多项运行时改进，适合评估 Deno 作为全栈脚本/构建环境的替代方案。

- [CSS vs JS 动画性能对比](https://www.joshwcomeau.com/animation/css-vs-javascript/)
  <br>对比 CSS 动画与 JavaScript 驱动动画在合成层、主线程阻塞、帧率等方面的性能差异，给出何时优先使用 CSS transition/animation 而非 JS 动画库的实践建议。

- [Chrome 预览 Declarative Partial Updates——支持乱序 HTML 流式插入更新局部区域](https://developer.chrome.com/blog/declarative-partial-updates)
  <br>Chrome 正在试验 Declarative Partial Updates 机制，服务端可声明式推送 HTML 片段并由浏览器流式合并到 DOM 指定位置，有望大幅简化"局部刷新+流式渲染"的实现成本。

- [Firefox 默认关闭 asm.js 优化——代码仍可运行，建议迁移至 WebAssembly](https://github.com/tc39/agendas/blob/main/2026/05.md)
  <br>SpiderMonkey 引擎默认禁用 asm.js 专项优化，存量代码不受影响仍可正常执行，但官方建议新项目及可维护的老代码逐步迁移至 WebAssembly 以获得更好性能与长期支持。

- [npm Staged Publishing 上线——npm stage publish上传包至暂存区，需维护者审批+2FA才正式发布（npm ≥11.15.0，pnpm 11.3也支持）](https://docs.npmjs.com/staged-publishing)
  <br>npm正式推出分阶段发布，包先进入暂存队列，须经维护者审批并通过2FA确认后才正式Registry生效，可显著降低供应链投毒风险；pnpm 11.3同步支持pnpm stage命令，适合CI中落地。

- [State of CSS 2026 调查开启——今年精简特性数，聚焦真正重要的CSS能力](https://survey.devographics.com/en-US/survey/state-of-css/2026)
  <br>2026年度State of CSS调查现已开放，今年对候选特性做了精简，重点征集开发者对真正重要CSS能力的反馈，是了解CSS新特性风向与参与社区标准讨论的好机会。

- [Google I/O 2026 HTML-in-Canvas API](https://www.webgpu.com/news/google-html-in-canvas-webgl-webgpu/)
  <br>2026 年 5 月 23 日，在 Google I/O 2026 上，Chrome 工程师 Thomas Nattestad 揭示了 HTML-in-Canvas API，这是一个新的 Web 平台功能。

- [CSS Container Queries 2026 实战](https://dev.to/zny10289/css-in-2026-container-queries-cascade-layers-and-the-end-of-utility-class-bloat-6k5)
  <br>2026 年 5 月 24 日发布，Container Queries、Cascade Layers 和现代选择器的组合消除了许多依赖 JavaScript 的样式需求。

- [2026 年 CSS 新特性](https://dev.to/armorbreak/css-in-2026-modern-techniques-you-might-not-know-2026-1lf7)
  <br>CSS 已经大幅演进，文章介绍了改变样式编写方式的现代特性。Container Queries（游戏规则改变者）、:has() 选择器、原生嵌套等特性让许多 JS 代码变得不再必要。

- [Chrome DevTools 149 新功能](https://developer.chrome.com/blog/new-in-devtools-149)
  <br>Chrome 149 带来 AI 辅助功能的重大升级，在 Application 面板中引入了实验性 WebMCP 调试工具，并扩展了代码覆盖范围。这些新功能将显著提升前端开发者的调试效率。

- [JavaScript ES2026 新特性详解](https://blog.webdevsimplified.com/2026-06/new-javascript-features-es2026/)
  <br>ES2025 已在浏览器中全面落地，ES2026 正在收尾，一些 ES2027 提案也已通过。文章详细介绍了每个值得了解的新 JavaScript 特性，包括显式资源管理、异步操作改进等，对前端开发者了解语言演进非常有价值。

- [Apache ECharts 6.1 发布——坐标轴 dataMin/dataMax、triggerEvent 等多处增强](https://github.com/apache/echarts/releases/tag/6.1.0)
  <br>ECharts 6.1 在坐标轴 min/max 支持取 dataMin/dataMax 自动计算、series.triggerEvent 细化等，方便制作自适应数据区间与交互捕获，建议图表项目跟进升级验证兼容性。

- [不滥用 aria-label 于 div / span——无障碍正确使用方式解析](https://www.matuzo.at/blog/2026/aria-label-generic-elements)
  <br>提醒开发者避免在无语义的 div/span 上直接加 aria-label，说明何时需配合 role 属性或使用原生语义元素，帮助规避常见的 ARIA 误用导致的屏幕阅读器失效问题。
